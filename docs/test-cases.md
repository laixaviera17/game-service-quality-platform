# 测试用例矩阵

下表对应仓库中可运行的 `pytest` 用例，重点覆盖奖励发放的幂等、事务边界、接口错误契约和数据质量定位。

| 场景 | 前置条件／输入 | 预期结果 |
| --- | --- | --- |
| 正常发奖 | 有效玩家、开启活动、库存充足 | 返回 `201`；库存减 1，玩家余额增加奖励值，写入成功记录 |
| 同请求重试 | 使用相同 `Idempotency-Key` 重复请求 | 返回历史结果并标记 `duplicated: true`；库存与余额不重复变化 |
| 幂等键冲突 | 同一个 Key 对应不同玩家或活动 | 返回 `409`，拒绝写入 |
| 库存不足 | 库存耗尽后发起新请求 | 返回 `409`；余额不再变化 |
| 不存在的资源 | 玩家或活动不存在 | 返回 `404` 及明确错误信息 |
| 缺少幂等键 | 请求头缺少 `Idempotency-Key` | FastAPI 参数校验返回 `422` |
| 重复发奖检查 | 同一玩家、同一活动存在多条成功记录 | `duplicate_reward` 返回失败计数与玩家／活动样本 |
| 异常数据检查 | 构造孤儿记录、非法状态、负余额 | 对应规则返回最多 3 条样本，便于定位 |

运行命令：

```bash
python3 -m pytest -q --cov=app --cov-report=term-missing
```

所有异常数据均由本地演示脚本构造，不来自外部或真实业务数据。
